# Owner B (GPU1) Implementation Plan

> 状态：已完成（存档）。不要再按本文执行；当前入口以 `README.md` / `notes/demo-runbook.md` 为准。

**Goal:** 在 `projects/4d-recon`（主阵地）打通 SelfCap（`bar-release.tar.gz`）到 FreeTimeGS 的 Gate-1 动态入口：生成 `images/ + sparse/0 + triangulation/*.npy`，并在 GPU1 跑出一段短训练视频与 `stats`，同时验证 `velocity` 非全 0。

**Architecture:** SelfCap `bar-release.tar.gz` 内含：
- `bar-release/videos/*.mp4`（多相机同步视频）
- `bar-release/optimized/{intri.yml,extri.yml}`（OpenCV YAML 格式内外参）
- `bar-release/pcds/%06d.ply`（逐帧稀疏点云，0..3539 连续）

实现一个最小 adapter：从 tar 中抽取 **8 个相机 + 60 帧**，用 OpenCV 解码视频输出 `images/<cam>/<frame>.jpg`，从 `pcds/` 输出 `triangulation/points3d_frame*.npy + colors_frame*.npy`，并用 `read_write_model.py` 生成参考 COLMAP `sparse/0/{cameras.bin,images.bin,points3D.bin}`（points3D 用首帧点云采样，确保 normalize 不会因空点云退化）。

**Tech Stack:** Python, OpenCV, NumPy, COLMAP model writer (`third_party/FreeTimeGsVanilla/datasets/read_write_model.py`), FreeTimeGsVanilla trainer

### Task B2: 实现 SelfCap -> FreeTime adapter（产出 images/sparse/triangulation）

**Files:**
- Create: `projects/4d-recon/scripts/adapt_selfcap_release_to_freetime.py`
- (Optional) Create: `projects/4d-recon/scripts/tests/test_selfcap_parsers.py`

**Step 1: 新建脚本骨架与 CLI（先跑通 help）**

要求 CLI（最少）：
- `--tar_gz`：`data/selfcap/bar-release.tar.gz`
- `--output_dir`：例如 `data/selfcap_bar_8cam60f`
- `--camera_ids`：例如 `02,03,04,05,06,07,08,09`
- `--frame_start`：默认 `0`
- `--num_frames`：默认 `60`
- `--image_downscale`：默认 `2`（输出分辨率减半，并同步缩放 K）
- `--max_points`：默认 `200000`（每帧点数上限，SelfCap 实际约 7k/帧）
- `--seed`：默认 `0`

Run:
```bash
cd /root/projects/4d-recon
PY=/root/projects/4d-recon/third_party/FreeTimeGsVanilla/.venv/bin/python
$PY scripts/adapt_selfcap_release_to_freetime.py --help | head
```

Expected:
- `--tar_gz` / `--output_dir` 出现在 help 中。

**Step 2: 实现 3 个核心模块（按这个顺序写，便于调试）**

1) `parse_opencv_yml_mats(text) -> dict[str, np.ndarray]`
- 解析 `!!opencv-matrix`：读取 `rows/cols/data:[...]` 并 reshape。
- 从 intri 取 `K_XX`；从 extri 取 `Rot_XX`、`T_XX`。

2) `read_binary_ply_xyz_rgb(fobj) -> (xyz[N,3] float32, rgb[N,3] float32)`
- 支持 SelfCap 的 `binary_little_endian`，属性顺序固定：`x y z red green blue`。

3) `write_colmap_sparse0(out_sparse_dir, camera_ids, K, Rot, T, points_xyz, points_rgb)`
- 用 `third_party/FreeTimeGsVanilla/datasets/read_write_model.py`：
  - Camera：`PINHOLE`，`params=[fx,fy,cx,cy]`（注意除以 `image_downscale`）
  - Image：每相机 1 张，`name=f\"{cam}.jpg\"`（flat 格式，stem 与 `images/<cam>/` 文件夹名对齐）
  - Point3D：从首帧点云采样（比如 5k~20k 点），`image_ids/point2D_idxs` 允许为空数组

Expected:
- `output_dir/sparse/0/{cameras.bin,images.bin,points3D.bin}` 三个文件都会生成。

**Step 3: 实现视频解码抽帧（OpenCV）**

实现 `extract_video_frames(video_path, out_dir, frame_start, num_frames, downscale)`：
- `cv2.VideoCapture(video_path)` 逐帧读
- 跳过 `frame_start`，保存 `num_frames` 帧到 `out_dir/%06d.jpg`
- 下采样：`cv2.resize(frame, (w//downscale, h//downscale))`

Expected:
- `output_dir/images/<cam>/000000.jpg` 存在且可读。

**Step 4: 实现逐帧点云导出 triangulation**

从 tar 读取 `bar-release/pcds/%06d.ply`，对 `frame_start..frame_start+num_frames-1`：
- 解析 PLY 得到 xyz/rgb
- （可选）按 `max_points` 随机采样
- 写入：
  - `triangulation/points3d_frame%06d.npy`
  - `triangulation/colors_frame%06d.npy`
  - `triangulation/frame_manifest.csv`

Expected:
- `triangulation/points3d_frame000000.npy` 到 `000059.npy` 共 60 个。

**Fallback（常见失败与对策）**
- 若日志出现 `After filtering: 0 Gaussians`：把 `--global-scale` 提高到 `12` 或 `20` 复跑。
- 若出现 `Singular matrix`（轨迹求逆退化）：保持 `--render-traj-path fixed`（不要用 arc）。
- 若显存/速度压力大：把相机数降到 6（删掉两个 `camera_ids`）或把 `num_frames` 先降到 30 做冒烟。
