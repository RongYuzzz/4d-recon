相关位置：`4D-Reconstruction.md:15`–`4D-Reconstruction.md:19`

### 2.2 “端到端”表述容易被抓漏洞

你说“不依赖外部 2D 分割/跟踪器作为前置模块”，这是好方向；但文中又包含“VGGT 特征提取→谱聚类→伪掩码”的独立流程（本质上是一个外置 preprocessing）。  
建议把“端到端”改成更可辩护的表述：**不依赖外部 2D 分割/跟踪器作为硬前置监督**，同时说明伪掩码属于“训练前/训练中生成的软监督”。

相关位置：`4D-Reconstruction.md:11`、`4D-Reconstruction.md:35`

### 2.3 关键技术缺“落地定义”，可行性存疑

1) **“全局注意力→跨帧像素对应矩阵”如何做**：  
像素级全连接对应是平方复杂度（O((HW)^2)），很容易在工程上不可用。需要至少写清一种可实现的稀疏化策略，例如：
- token/patch 级别注意力；
- top‑k 稀疏对应；
- 只在关键点/可见区域上建立对应；
- 或者以 tracklet（如 KLT）为载体，注意力仅用于加权/筛选。

相关位置：`4D-Reconstruction.md:38`

2) **“注意力引导对比损失”施加对象不明**：  
你写“渲染场中对应的高斯基元特征必须相似”（`4D-Reconstruction.md:39`），但没有定义：
- 高斯“特征”是什么（颜色/隐变量/渲染出的 feature map）？
- 像素/patch ↔ 高斯如何绑定（投影采样、可微聚合、或以渲染 feature map 表示）？
- 对比学习的正负样本构造是什么？

如果这三点不写清，同行会默认“概念正确但不可实现/不可复现”。

相关位置：`4D-Reconstruction.md:39`

### 2.4 预期目标过“硬”，评测口径不自洽

- “mIoU 优于依赖外部先验的方法”（`4D-Reconstruction.md:23`）会被追问：  
  - 你的分割 GT 来自哪里？哪些数据集提供？  
  - 你要对标哪些方法？这些方法往往更强，你凭什么必然超过？
- 建议把目标改为更可答辩的格式：  
  - **对标对象**（segmentation-free vs 使用外部先验的 upper bound）  
  - **主指标**（建议加入时序稳定性指标，如 tLPIPS/temporal consistency）  
  - **“力争/预期”而非“保证优于”** 的措辞

相关位置：`4D-Reconstruction.md:21`–`4D-Reconstruction.md:25`

### 2.5 缺“算力预算/止损/Plan‑B”，风险不可控

开题的步骤计划（`4D-Reconstruction.md:50`–`4D-Reconstruction.md:56`）是典型理想化排期，但没有写：

建议增加 **“资源约束与止损策略”** 小节，明确 MVP 最低交付与时间盒。
